Code Documentation Generator

A local-first "Code-to-Doc" generator using Few-Shot Prompting with a lightweight LLM (Qwen2.5-1.5B-Instruct) to generate structured, validated Python docstrings — entirely offline.

The Challenge

Developers frequently neglect documentation, accumulating technical debt. Existing tools either hallucinate parameter names or depend on expensive cloud APIs.

Engineering Solutions

  • Structured Output Reliability: Implemented a JSON-enforced prompt strategy that eliminates conversational output, forcing the model to return strictly parsable schemas only.
  • Hallucination Prevention: Built a heuristic AST validator that cross-references generated parameter names against actual source code — any non-existent argument is automatically rejected.
  • Evaluation Pipeline: Developed a custom evaluation engine computing BLEU, ROUGE-L, and METEOR scores against human-written reference documentation for quantitative quality benchmarking.

Hardware & Performance

  • Requirements: Modern CPU (4+ cores) with 8-12 GB RAM; GPU not required.
  • Runtime Estimate: First run downloads the model (~30-90s). Afterward expect ~5-20s per file, depending on size.

Tech Stack

Python 3.11, Qwen2.5-1.5B-Instruct, DistilGPT2, HuggingFace Transformers, PyTorch, NLTK, tiktoken, Pydantic, Streamlit
